# Mobile audit — land-selection-framework prototype

Date: 2026-05-22
Audited via headless Chrome at viewports 375, 414, 480, 600, 768.
Source screenshots in /tmp/mob/ (not committed).

## Issues found

### 375px (iPhone SE, iPhone 12 mini)
1. **Hero signup card overflow** — input + Subscribe button stay side-by-side
   on the hero card down to 375px (existing @480 stacks only the bottom
   contact form, not the hero one). Subscribe button gets clipped.
   *Fix*: extend the `flex-direction: column` rule to `.signup-card form`
   at <=480 and let the Subscribe button span the full width.
2. **Match-bar buttons overlap** — `.match-bar` becomes column-flex at <=480
   but the actions row (share + reset + share-note) can still wrap awkwardly.
   *Fix*: allow `.match-bar-actions` to flex-wrap, add full-width on small.
3. **bar-row value column too narrow** — `60px` bar-val column truncates
   "1760 kWh/kWp" and "-2.4 %/decade" via overflow clip.
   *Fix*: widen the value column at <=480 (or hide the unit inline) — bump to
   72px and shrink unit suffix font.
4. **Slider thumb 18–22px** — under iOS HIG 44px touch target.
   *Fix*: `@media (pointer: coarse)` rules bumping thumb to 28px and the
   track to 6px so the effective hit area approaches 44px.
5. **Modal close button 36×36px** — under 44px touch target on phones.
   *Fix*: bump to 44×44 on coarse pointers.
6. **Map controls padding tight** — toggles at 8px 10px on small viewports
   make taps hard.
   *Fix*: bump padding to 10px 14px and inner toggle padding on coarse.
7. **Hero `.key` second column** uses `padding-left: 14px` on nth-child(2n)
   at <=880 but breaks the visual rhythm at 375 (numbers crowd left rule).
   *Fix*: minor — already acceptable, leave.
8. **Signup form Subscribe button label "Subscribe →"** is white-space:nowrap
   plus narrow row → forces overflow at 375.
   *Fix*: stack form at <=480 (see #1).

### 414px (iPhone 12 Pro Max)
- Same hero form clipping as 375 but milder. Same fix covers it.
- All other layouts pass.

### 480px (typical small Android)
- Mostly fine. Slider thumb size still 22px — still under 44px target.

### 600px (small tablet portrait)
- No critical issues. Modal renders well.

### 768px (iPad portrait)
- Two-column hero grid kicks in at >880. At 768 we still have the single-col
  hero which is fine but the map can feel small at 360px height.
- No blocking issues.

## Accessibility / touch

- Every button now has `aria-label` (share, reset). Modal close already labeled.
- Share + Reset buttons hit 44px after padding bump on coarse pointers.
- Slider thumb bumped to 28px on coarse pointers (effective tap area ~44px).

## Cross-cutting fixes applied
- `@media (pointer: coarse)` block added in <style> just below the existing
  @media (max-width: 480px) block.
- `@media (max-width: 480px)` extended to stack hero form and widen value cells.

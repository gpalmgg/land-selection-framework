## HIGH

### V-H1 — Modal copy contradicts the interaction-lock mechanism
- **Where:** `index.html` modal footnote ("The rest of the page stays open either way") vs CSS `body.interaction-locked` (lines ~819-828) which sets `pointer-events:none; opacity:0.42; filter:grayscale` on `.criteria .slider-block` and `.reset`.
- **What:** The modal verbally reassures the visitor the page stays usable, but a lock state exists that disables the core threshold sliders. If that lock is applied while the modal is shown (growth agent is tracing the exact trigger in `main.js`), the central interaction is dead behind an email ask — on a cold, high-traffic audience.
- **Why it matters:** Highest-leverage bounce risk on the whole site. People came to *play with the sliders*; gating that is the fastest way to lose them.
- **Fix:** Reconcile copy and behavior — either never lock the sliders (recommended for a high-traffic top-of-funnel), or make the copy honest. Coordinate with growth-axis recommendation.
- **Effort:** S (once the trigger is located).

### V-H2 — RETRACTED (false positive). Was: "West Cork vs Connemara inconsistency."
- **Verification:** `data/regions.js:40` name = `'West Cork / Connemara'` (a deliberately combined western-Ireland candidate); `deeper.html:487` case study = "West Cork & Connemara"; footer CTA "Connemara" is shorthand for the same region. **No inconsistency — this is intentional and coherent.** What I observed in the bars was truncation of the long combined name (see V-M4 below), not a naming conflict. Verified before asserting; retracted.

## MEDIUM

### V-M1 — Mobile map control panel occludes ~half the map
- **Where:** `.map-controls` at ≤880/≤480 breakpoints, observed at 390px.
- **What:** The Layers panel (6 toggles + terrain) covers a large portion of the small-screen map, partly hiding markers/geography.
- **Why it matters:** Map is a credibility centerpiece ("real data layers"); on mobile it's half-covered by chrome.
- **Fix:** Collapsible/disclosure control on mobile (tap "Layers" to expand), or move toggles below the map on small screens.
- **Effort:** M.

### V-M2 — Criterion numbering starts at "#2"
- **Where:** First criteria card renders "Climate trajectory **#2**".
- **What:** Unclear whether a #1 exists / whether ordering is intentional or an artifact of the expansion.
- **Why it matters:** Numbered criteria that don't start at 1 read as a bug.
- **Fix:** Verify the `criteria` array order/numbers in `regions.js` (content agent reads this); make sequential or remove visible numbers.
- **Effort:** S.

### V-M3 — Stale "four" copy (visual manifestation of the 4→10 expansion)
- **Where:** hero stat, lede sentence, regions-intro heading — visible on BOTH desktop and mobile.
- **Why it matters:** First-impression contradiction (says four, shows ten). Most-embarrassing-while-live item.
- **Fix:** Content agent owns the exhaustive cleanup list; trivial string edits.
- **Effort:** S.

## LOW / NOTES

### V-M4 — Long combined region names truncate in criteria-bar labels
- **Where:** `.bar-row .reg` (overflow:hidden + ellipsis) at the 90px (desktop) / 60px (≤480px) name column.
- **What:** "West Cork / Connemara", "Pembrokeshire / West Wales", "Cévennes / Languedoc", "Saxony-Anhalt rural" render as "West Cor…" etc. in the per-criterion bar charts.
- **Why it matters:** Minor legibility loss on the combined-name regions; not a credibility issue.
- **Fix:** Widen the name column, shorten bar labels (short name in bars, full name in grid/table), or allow 2-line wrap.
- **Effort:** S.
